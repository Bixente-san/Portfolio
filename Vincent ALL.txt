Focus sur DatAOdesign

Contexte du projet :
Remplacement d'un ancien outil Power BI pour la visualisation des données de validations télébillettiques.
Besoin critique des designers d'offre d'accéder aux données de validation de bus pour optimiser l'offre de transport en vue des RAO.
Objectif : offrir un accès à des données déjà transformées et rapidement exploitables.
Projet stratégique dans le contexte d'ouverture à la concurrence, avec une vocation d'amélioration.

Résultats clés :
10 utilisateurs formés.
19 Go de données traitées.
500 heures de travail.

Architecture du projet:
Extraction des données depuis le SIDV.
Pré-traitement des données en Python (plus de 400 lignes de code !).
Construction de l’application sur QlikSense.
Ajustements.
Présentation et déploiement aux utilisateurs.

À l’intérieur du modèle de données:
Les données sont intégrées dans Qlik via un script de chargement.
Un modèle de données organise et relie les différentes sources pour permettre une visualisation cohérente et exploitable.
Exemple :
Table : CALENDAR.
Données de validation traitées : CODE_LIGNE, CODE_JOUR, LOT.
Structure et jointures :
Flag rémunérations par jour (Flag Remuneration Par Jour).
Catégorie rémunération (Catégorie Remuneration).
Type design (Type Design).
Processus global :
Les données sont nettoyées et prêtes avant d’être intégrées pour une visualisation sur QlikSense.
